# SQL AI Agent — Backend

## Overview

The **SQL AI Agent Backend** is a REST API that enables users to query relational databases using **natural language**. The system uses an AI model to translate user questions into SQL queries, executes them securely against the database, and returns a human-readable explanation of the results.

The backend acts as a **secure intermediary layer** between users and the database, ensuring that AI-generated queries follow strict validation and access control rules before execution.

---

# Key Features

### Natural Language to SQL

Users can ask questions in plain English and the system automatically generates the appropriate SQL query.

Example prompt:

```
Show all posts created in the last week
```

The AI agent converts this into a SQL query and retrieves the data.

---

### Secure Query Execution

Before execution, generated SQL queries pass through a **validation layer** that ensures the query is safe and allowed for the user’s role.

This prevents unintended database modifications and protects against destructive queries.

---

### Role-Based Access Control (RBAC)

Access to database operations is restricted based on user roles. This ensures users only perform operations appropriate to their permission level.

---

### AI-Powered Result Explanation

After executing the SQL query, the backend sends the database response back to the AI model, which generates a **clear explanation of the result in natural language**.

# Authentication

The API uses **JWT-based authentication**.

Users must authenticate and provide a **Bearer token** in the request header.

Example header:

```
Authorization: Bearer <JWT_TOKEN>
```

---

# Roles and Permissions

The system defines three roles with different database access levels.

| Role       | Description     | Allowed Operations               |
| ---------- | --------------- | -------------------------------- |
| USER       | Standard user   | Read-only queries                |
| ADMIN      | Elevated access | Read + limited data modification |
| SUPERADMIN | Full access     | All SQL operations               |

The validation layer ensures that queries generated by the AI comply with these permissions.

# API Endpoints

## Ask SQL Agent

Allows users to ask a natural language question that will be converted into a SQL query.

### Endpoint

```
POST api/Agent/ask
```

### Full URL

```
http://localhost:8080/api/Agent/ask
```

---

### Authorization

Required

```
Authorization: Bearer <JWT_TOKEN>
```

---

### Request Body

```json
{
  "prompt": "Show the latest 10 posts"
}
```

---

### Example Request

```http
POST /api/Agent/ask HTTP/1.1
Host: localhost:8080
Authorization: Bearer YOUR_TOKEN
Content-Type: application/json

{
  "prompt": "List all users created this month"
}
```

---

### Example Response

```json
{
  "executionTime": 1.24,
  "result": "There are 42 users created this month. The most recent user joined on March 9.",
  "status": true,
  "message": "Query successfully processed",
  "errors": null
}
```

---

# Request Processing Flow

The backend processes requests through the following pipeline:

1. **User submits a natural language prompt**
2. The system retrieves the **database schema**
3. The AI model generates a **SQL query**
4. The query passes through **security validation**
5. The validated query is executed against the database
6. The database response is sent back to the AI model
7. The AI generates a **human-readable explanation**
8. The final response is returned to the user

---

# Project Architecture

The backend follows a modular service-based architecture:

* **Agent Service**
  Communicates with the AI model to generate SQL and explanations.

* **Schema Service**
  Retrieves the database schema to guide AI query generation.

* **SQL Service**
  Executes validated queries against the database.

* **SQL Validator**
  Enforces role-based query restrictions.

* **API Controller**
  Handles request processing and orchestrates the workflow.
